Translating Photographer.io

Jul 9th, 2013

Thanks to some hard-working and enthusiastic volunteers, work has already begun on translating the application into a variety of other languages. Anyone can join in and help translate the site into their favoured language!

I’m incredibly grateful to all the effort the translators are putting into this. I built the application from the start with an international audience in mind, and in theory enabling multi-lingual support is straight-forward once the translations are complete. The ability to switch the site language is planned for the next couple of weeks.

If you want to help out then head on over to the Transifex page. If your language doesn’t have an active project yet then be the first to create it!

We’re now Open Source!

Jul 8th, 2013

As of yesterday Photographer.io is now open source. I’ve been meaning to release the source code for quite a while and it’s a great relief to finally get it out there, even if it does reveal the currently woeful test suite. You can find the source on GitHub under an MIT license.

There are a number of reasons why I wanted to open source this application:

I’d love to get other people involved in creating a great photography community.

Opening up the source code to your contributions is really exciting to me. I’ve always been open to suggestions and ideas (and the vast majority of the open issue tickets on GitHub are suggestions from members), and this will hopefully let others contribute in a meaningful way.

It relieves some of the pressure on being the only developer on the project.

If something were to happen to me, the project could still continue. I’d hate to leave everyone in the lurch in the case of my unexpected demise.

It will improve the quality and security of the site by opening the code to peer review.

I care a lot about security and code quality, but talking about security and testing those claims are two very different subjects. If you find a security flaw then please get in touch at security@photographer.io.

It’s great to finally have a project worth open sourcing.

On a more selfish note; I’m really excited to have something I feel is worth open sourcing. I’ve spent 2 months working nearly every evening to get the site to a stable point and I’m generally pleased with where it is (other than the dire test suite). There will always be a part of me that feels a little apprehensive about offering up the code I’ve spent so much time on recently, but I’m confident that this is the right thing to do.

Getting involved

There’s lots to do, including documentation and testing, but it’s hopefully in a state where you can at least look through the code and get an understanding of what’s going on. If you have any questions then please ask away. I’ll make sure to put up some notes on how to contribute shortly.

Username Support

Jun 8th, 2013

Usernames are now live! Finally, you can access your public profile via https://www.photographer.io/u/robotmay rather than /users/1.

To facilitate this change I’ve added a new ‘username’ field, which must be a unique, lowercase name containing only letters, numbers, and underscores. This will then be used in URLs across the site. Your first username has been automatically generated from your current display name, but you can change this (up to 2 times) from your account settings page.

Accessing a page via an old username will redirect to your new one. In the future old names will be freed up for new users to take them after a certain length of time.

If you have any questions about how this system works; get in touch!

DNS Resolution Problems

Jun 3rd, 2013

For the past hour our DNS host (DNSimple.com) has been suffering a large Distributed Denial of Service attack on its nameservers. They’ve managed to get basic service back up, but a component that is used to point the root domain (photographer.io) is currently disabled. I’ve modified the DNS settings for now so that www.photographer.io still resolves correctly, though the root domain will remain down until they’ve fixed this issue.

My apologies for the downtime; I’ll be looking for a way to prevent this causing such a detrimental effect in future.
